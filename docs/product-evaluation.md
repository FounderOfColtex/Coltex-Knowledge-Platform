# Zypher Product Evaluation Guide

The product includes benchmark datasets and an evaluation pipeline that produces **evidence** that the knowledge base improves AI retrieval performance.

## Datasets

Generated by `make benchmarks`:

### FAQ pairs (`benchmarks/faq_pairs.jsonl`)

```json
{
  "id": "faq-CHUNK-00000",
  "question": "What is Retrieval-Augmented Generation (RAG) for code?",
  "expected_doc_ids": ["CHUNK-00000"],
  "expected_keywords": ["retrieval", "rag", "knowledge", "hallucinations"],
  "category": "chunked",
  "tags": ["faq", "rag"]
}
```

### Retrieval gold (`benchmarks/retrieval_gold.jsonl`)

```json
{
  "query": "What is Retrieval-Augmented Generation (RAG) for code?",
  "relevant_doc_ids": ["CHUNK-00000", "CHUNK-00001", "..."],
  "difficulty": "easy"
}
```

### RAG eval (`benchmarks/rag_eval.jsonl`)

Includes rubric and expected keywords for faithfulness checks.

## Metrics

Run `make evaluate` to produce `benchmarks/evaluation_report.json`.

| Metric | Definition | Target |
|--------|------------|--------|
| `recall@k` | Fraction of relevant doc IDs found in top-k results | ≥ 50% |
| `context_keyword_overlap` | Share of expected keywords present in retrieved context | Informational |
| `metadata_accuracy` | Documents with valid `id` and `title` | ≥ 90% |

### Recall@k

```
recall@k = |retrieved[:k] ∩ relevant| / |relevant|
```

Computed over all queries in `retrieval_gold.jsonl` using the curated Zypher Brain (`config/brain_curated.yaml`).

### Context keyword overlap

For each FAQ question, retrieves context and checks how many expected keywords appear. This proxies **faithfulness** — whether the retrieved context contains the facts needed to answer correctly.

### Metadata accuracy

Validates that every curated document has non-empty `id` and `title` fields.

## Evidence report

Example output in `benchmarks/evaluation_report.json`:

```json
{
  "passed": true,
  "evidence": [
    "Retrieval recall@8: 72.50% (target ≥ 50%, PASS)",
    "Context keyword overlap: 68.20% across 12 FAQ queries",
    "Metadata accuracy: 100.00% (target ≥ 90%, PASS)"
  ]
}
```

## Interpreting results

- **Recall below target** — improve document titles, tags, or graph links; add missing FAQ coverage
- **Low keyword overlap** — chunking may be splitting answers; adjust `chunking` config
- **Metadata failures** — fix frontmatter in source documents before release

## Reproducibility

1. `config/product.yaml` pins evaluation thresholds
2. `config/brain_curated.yaml` pins knowledge scope
3. `data/product/manifest.json` records artifact checksums
4. Evaluation uses `sentence-transformers/all-MiniLM-L6-v2` (same as production brain)

## CI integration

```bash
make product && make evaluate
```

Exit code is non-zero if quality gates or evaluation thresholds fail.
